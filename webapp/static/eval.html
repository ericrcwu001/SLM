A model does not pass because its output looks plausible. It passes because the entire artifact survives a measurable chain of obligations.
01

Test the product behavior, not the training loss.

The harness starts from the client-visible contract: read a request into an AttributeSpec, route it as grade, clarify, or refuse, produce a valid transform, preserve image integrity, and decline what a global LUT cannot solve.

02

Make every failure attributable.

Boundary, syntax, decode, color direction, target fidelity, safety, and style are separate layers. A failure says what broke, not merely that a score went down.

03

Challenge the need for fine-tuning.

The SFT checkpoint must beat nulls, constants, deterministic recipes, prompt-only ablations, image ablations, and frontier prompting — otherwise the claim is narrowed.

04

Separate diagnosis from shipping.

Small slices and qualitative examples illuminate behavior. Only predeclared, adequately powered metrics can approve a release.

05

Be honest about the holdout.

The split is unit-aware, so no near-duplicate LUT straddles train and test; every held-out row is scored, not a convenient sample; and a supported row counts only with exactly 64 code positions. Where the frozen decoder is off, honesty slices use decoder-free proxies rather than pretending to measure color.

The deterministic stack

Nine lenses. One non-negotiable verdict.

Supported rows must clear every deterministic layer from L0 through L7. The L8 judge adds explanation and qualitative context, but it cannot rescue an invalid or unsafe transform. In v1 the frozen decoder stays disabled, so the decode-dependent layers (L2, L4–L7) run as decoder-free proxies or are owner-gated; the load-bearing spine is L0 boundary, L1 syntax, the routing metrics, and held-out token accuracy.

L0
Scope

Boundary & route

The interpreter's route is scored three ways. A grade must not be refused; a refuse passes only with the exact token <unsupported>, split into out_of_scope and out_of_gamut; clarify is its own class.

No partial compliance. No friendly prose.
L1
Form

Syntax

A supported output is exactly LUT-BOS, 64 valid code tokens, and LUT-EOS. Runtime constrained generation should make syntax failures implementation bugs.

Free generation measures learned discipline.
L2
Artifact

Decode & export

The 64 codes decode through the frozen VQ decoder into a finite canonical 17×17×17 residual LUT and a valid pinned .cube export.

Domain, grid, hashes, and interpolation must match.
L3
Compression

Tokenizer gate

The frozen tokenizer must satisfy reconstruction quality across the mean, difficult tail, source families, and target classes before model scores are trusted.

Bad compression cannot masquerade as bad SFT.
L4
Intent

Direction

Measured behavior_v2 must move in every requested direction — warmer, cooler, brighter, contrast toe and shoulder, matte, a global hue cast, per-tone-region hue, and per-hue saturation across seven sectors — within calibrated magnitude windows.

Intent is checked numerically, not by label matching.
L5
Similarity

Target fidelity

Depending on the row, the decoded LUT must match an exact target, one of several valid references, or a declared behavior window using image and chart ΔE00.

Underspecified requests get multiple valid answers — not a fake single truth.
L6
Integrity

LUT safety

Clip rate, out-of-range values, smoothness, foldover, neutral drift, and skin-locus behavior are tested before a transform can pass.

A pretty average cannot hide a dangerous tail.
L7
Semantics

Style recipe

Named styles must land inside recipe windows and remain discriminable from neighboring styles rather than collapsing into one generic warm filter.

Correct family, correct margin, no multi-match.
L8
Context

LLM / VLM judge

A pinned judge records perceptual and instruction-alignment commentary for review. It is evidence, not authority.

L8 cannot override L0–L7.
The decoder stays frozen and off in v1.
L2 and L4–L7 depend on decoding 64 codes into a LUT. Until the frozen decoder is owner-enabled, those layers run decoder-free proxies, and the training loop is steered by held-out, teacher-forced LUT-code token accuracy — the faithful-ish proxy that needs no decode, since the 64 codes deterministically decode to the target residual LUT.

Frozen evaluation design

One headline number, many ways to break it.

The aggregate pass rate answers “does it work?” The slices answer “where, for whom, and under what conditions does it stop working?”

usage_weighted_headline

The client-facing mix of common supported requests, reserved for the primary quality claim.

image_sensitivity

Identical text across different images where the correct LUTs must differ. Defends against a prompt-only shortcut.

hard_cli_inputs

Phone exports, screenshots, heavy JPEG compression, odd white balance, ICC variation, and size extremes.

subtle_control

Small but visible edits that expose a model that only knows exaggerated transformations.

style_discriminability

Single-style rows with neighbor-exclusion tests, recipe windows, and confusion analysis.

expert_holdout

Held-out PPR10K and FiveK expert identities absent from the SFT training set.

unseen_family

Held-out source, style, or LUT families that measure generalization beyond memorized packs.

interpreter

The interpreter's own scores: attribute-F1 against the gold AttributeSpec, three-way route accuracy, and over-refusal on valid work.

refuse_out_of_scope · out_of_gamut

Non-global edits versus global looks the frozen tokenizer cannot represent — each refusal family measured on its own.

paraphrase · named · nonce concept

Decoder-free honesty: paraphrases stay consistent, known styles map to real axes, and invented terms route to clarify or refuse instead of hallucinating.

unsupported + boundary_pairs

Local edits, geometry, generation, reference transfer, mixed traps, and contrastive near-boundary pairs.

Predeclared release criteria

A model ships on confidence bounds, not point estimates.

Every gated metric has a frozen slice, minimum sample size, direction of the bound, test family, and underpowered behavior. The current v1 headline floor is 350 rows because the reserve is approximately 382 eligible supported rows.

Roadmap · not done yet

Next step: GRPO.

Planned · needs more A100 time + compute · no shipped result

The dashboard above is honest about the gap: free-running greedy generation collapses to ≈0.16 behavioral fidelity (~94% collapse), while sample-and-rerank best-of-N already reaches ≈0.42. The next step is GRPO — on-policy RL to bake that covered-but-unselected quality into a single deterministic greedy pass. It is not run yet; it needs GPU time and may need knob tuning. Here is why the bet is worth making — and where it could fail.

◆ The strongest evidence — from our own data

Why it should work

Coverage already exists 0.42oracle@320.42best-of-N
The good trajectories are already in the policy's sampling distribution — greedy just doesn't pick them. GRPO only has to shift mass toward behavior the model already produces, not discover new capability. That is the easy case for policy gradient.
High within-prompt variance = strong learning signal Sample-t0.7 mean fidelity is only ~0.09, but the best of 32 is ~0.42. That heavy right tail gives each group of rollouts a wide reward spread — exactly what group-relative advantage exploits (it pushes toward the top of each group). Low mean + high max is the favorable signature, not a red flag.
The capability is there Teacher-forced fidelity is 0.708 — given a good prefix the model renders the spec fine. The failure is purely free-running collapse (the GRPO run initializes right here: greedy ≈0.16, collapse >90%). On-policy RL trains on the model's own rollouts — the textbook fix for exactly this exposure-bias mismatch.
Our own gate points here The oracle_at_n rule: ≥0.6 → distill · 0.3–0.6 → capped · ≤0.3 → escalate. At 0.42 we're in the capped band — best-of-N ships today, distillation alone is capped, so RL/GRPO is the escalation of record.

◇ The honest counter-case

Where it could fall short

RL sharpens more than it invents The realistic target is greedy climbing toward ~0.42 — best-of-N parity, but in one deterministic pass — not blowing past it. Reaching 0.42 is the win; 0.7 is not expected.
Reward hacking is real The reward only scores requested axes, so the policy can game them (dominant-code collapse, ΔE drift). That is why the collapse penalty + guard panel + KL-to-P6 exist. If it hacks, ΔE/collapse rise — and BEST won't promote it.
Movement may be weak or slow 3B QLoRA, lr 5e-6, μ=1 — a conservative REINFORCE+KL. It might need more than 500 steps, or knob tuning.
Literature prior GRPO/RLHF in the pass@k ≫ pass@1 regime (DeepSeekMath / R1 and the broader RLHF literature) reliably raises greedy/pass@1 toward pass@k by sharpening onto high-reward modes already sampled — precisely greedy 0.16 vs oracle 0.42.
↧

Why the bet is cheap

BEST only promotes a guard-clean fidelity improvement. Worst case, GRPO doesn't help and BEST stays at the ~0.16 init — you've spent GPU time, not shipped a regression. You cannot end up worse than P6.
